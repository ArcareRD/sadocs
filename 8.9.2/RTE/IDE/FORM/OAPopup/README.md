### <div id="valid">執行條件</div>

### <div id="query">視窗內容</div>
按下 <rte>系統功能鍵.查詢視窗</rte>，會開啟對應的功能。

### <div id="form">開啟表單 <path>(視窗內容)</path></div>
開啟表單，選取記錄帶回。<ic>(FW0000002946)</ic>
* 開啟模式
    * 於PC上，開啟新視窗。
    * 於行動裝置上，Frame開單。
* 若<輸入值>有值時，會駐留在至符合條件的記錄上 (依元件對應的勾選帶回_來源欄位)。
    * 若為文字時 : 會找開頭=元件值的記錄。
    * 其餘型態，則須完全相同。
* 挑選指定記錄後，可擇其一，帶回資料。
    * 執行<rte>系統功能鍵.回傳勾選的記錄</rte>
    * `多筆瀏覽`元件執行<rte>雙擊</rte>
    * `多筆表格`元件執行<rte>雙擊</rte>

### <div id="form_form">表單名稱 <path>(視窗內容\開啟表單)</path></div>
開啟的表單。
### <div id="form_filter">過濾條件 <path>(視窗內容\開啟表單)</path></div>
開啟表單後，是否過濾部份資料。
* 可空白

<ps>注意</ps> 若開啟表單_<rte>主檔區</rte>的資料來源改變時，須重新設定。

### <div id="form_param">傳遞參數 <path>(視窗內容\開啟表單)</path></div>
開啟表單的傳遞參數。<ic>(FW0000002450)</ic>
* 可不設 (但開啟表單要控制好，沒傳遞的處理。)

### <div id="form_param_name">參數名 <path>(視窗內容\開啟表單\傳遞參數)</path></div>
傳遞參數名。

<ps>注意</ps> 當`參數名`改變時，本單也要改版。

### <div id="form_param_sourcetype">參數類別 <path>(視窗內容\開啟表單\傳遞參數)</path></div>
傳入值的類別。
* `表單元件`
* `固定值`
* `參數`

### <div id="form_param_source">參數體 <path>(視窗內容\開啟表單\傳遞參數)</path></div>
傳入值。
* 依`參數類別`
    * `表單元件`
        `表單元件` / `隱藏表單元件`。
    * `固定值`
        * 依`接收參數`的`型態`，輸入格式不同
            * 文字 : 不須加上雙引號(")。
            * 數字
            * 日期 : 待測
            * 邏輯 : true / false。
    * `參數`
        `表單`的`接收參數`。

### <div id="form_choosetype">筆數 <path>(視窗內容\開啟表單\勾選限制)</path></div>
可帶回的筆數。
* 單選
    * 勾選帶回後，自動執行檢錯存後(#7781)。
* 複選 <ic>(FW0000002982)</ic>
    * 限為`多筆表格`的子元件，且`表單類型`為`單頭一身多筆型（雙檔）` / `多頭單身多筆型（雙多筆）`，且依附`多筆表格`的`存在檔區`=`表身`。
    * 帶回資料時，第一筆會固定覆蓋駐留筆。第二筆以後會插入新的記錄在駐留筆之後。

### <div id="form_onlychoose">限定勾選 <path>(視窗內容\開啟表單\勾選限制)</path></div>
是否只能是開窗挑回，不可自行輸入。
* `筆數`
    * `單選` <ic>(FW0000006440)</ic>
    * `複選` <ic>(FW0000002665)</ic>
* 若非開窗挑回時，於<rte>元件跳離</rte>時，顯示錯誤訊息"※系統通知※ 不允自行輸入，只能開窗挑回！"。

### <div id="form_appendinit">新增初始值 <path>(視窗內容\開啟表單\勾選限制)</path></div>
帶回的資料，於新增`多筆表格`的記錄時，是否執行新增初始值。
* 當`筆數`為`複選`時，才可設定。

### <div id="form_put">勾選帶回 <path>(視窗內容\開啟表單)</path></div>
定義帶回欄位要寫至哪個元件。
* 若帶回多個欄位時 <ic>(FW0000006440)</ic>

### <div id="form_put_ctrl">接收欄位 <path>(視窗內容\開啟表單\勾選帶回)</path></div>
寫至哪個元件。

### <div id="form_put_sourcefield">來源欄位 <path>(視窗內容\開啟表單\勾選帶回)</path></div>
帶回欄位。

### <div id="form_remove_duplicates">剔除已存 <path>(視窗內容\開啟表單\勾選帶回)</path></div>
若帶回的資料，已存在`多筆表格`時，該筆則不帶回。
* 當`筆數`為`複選`時，才可設定。

### <div id="form_run_relation_update">主動更新 <path>(視窗內容\開啟表單\勾選帶回)</path></div>
是否執行該`元件`的`更新相關值` / `被動更新`。

### <div id="form_copy">複寫資料 <path>(視窗內容\開啟表單)</path></div>
預設是否帶入`多筆表格`駐留筆資料。<ic>(FW0000001821)</ic>

### <div id="form_copy_ctrl">欄位 <path>(視窗內容\開啟表單\複寫資料)</path></div>
預設帶入的元件。

### <div id="folder">資料夾 <path>(視窗內容\檔案總管\型態)</path></div>
開啟檔案總管，挑選資料夾。<ic>(FW0000002897)</ic>
* 開啟介面
    * STD表單

        ![STD_資料夾]

        * 視窗位置 : 表單區的正中間。
        * 上傳時，下方會出現單一檔案上傳進度。
    * RWD表單

        ![RWD_資料夾]

        * 視窗位置 : 表單區的上方。
        * 上傳時，下方會出現單一檔案上傳進度。
* 挑選後，按上傳
    * 上傳至Server端。
        * 上傳檔案會暫存在 : <rte>Site資料庫設定</rte>的<rte>共用資料夾實體路徑</rte>的資料夾裡。    
	        * 檔案會透過<rte>系統排程</rte>刪除。
    * 關閉介面。
    * 元件顯示資料夾名稱。
    * 元件的`檢視表對應欄位`內容會記錄於Server端的暫存資料夾位置。但在<rte>系統功能鍵.存回</rte>後，會僅剩資料夾名稱。

### <div id="file">檔案 <path>(視窗內容\檔案總管\型態)</path></div>
開啟檔案總管，挑選檔案。<ic>(FW0000002634)</ic>
* 開啟介面
    * STD表單

        ![STD_檔案]

        * 視窗位置 : 表單區的正中間。
        * 上傳時，下方會出現單一檔案上傳進度。

    * RWD表單

        ![RWD_檔案]

        * 視窗位置 : 表單區的上方。
        * 上傳時，下方會出現單一檔案上傳進度。

* 挑選後，會立即上傳。
    * 上傳至Server端。
        * 若副檔名為 JSP / HTM / HTML / JS / EXE / DLL / BAT 時，會彈出錯誤訊息盒"不安全的檔案類型"			
        * 若檔案大小 = 0 時，會彈出錯誤訊息盒"挑選的檔案大小為 0 Bytes，不允上傳！"			
        * 若檔案大小 > [系統上傳檔案大小]時，會彈出錯誤訊息盒"檔案不可大於 %1 MB!"			
	        * 註.系統上傳檔案大小 : Site 企業設定 → 企業資料 → 系統_上傳檔案大小		
        * 上傳檔案會暫存在 : <rte>Site資料庫設定</rte>的<rte>共用資料夾實體路徑</rte>的資料夾裡。
	        * 檔案會透過<rte>系統排程</rte>刪除。
    * 關閉介面。
    * 元件顯示檔案名稱。
    * 元件的`檢視表對應欄位`內容會記錄於Server端的暫存資料夾位置。但在<rte>系統功能鍵.存回</rte>後，會僅剩資料夾名稱。


### <div id="file_choosetype">筆數 <path>(視窗內容\檔案總管\型態\檔案)</path></div>
可選取的檔案個數。
* 單選
* 複選
    * 限為`多筆表格`的子元件，且`表單類型`為`單頭一身多筆型（雙檔）` / `多頭單身多筆型（雙多筆）`，且依附`多筆表格`的`存在檔區`=`表身`。
    * 若元件不是`多筆表格`的子元件時，會出現錯誤訊息盒"上傳多個檔案需在多筆元件內才可執行"。


### <div id="file_binary_ctrl">檔案儲存欄位 <path>(視窗內容\檔案總管\型態\檔案)</path></div>
儲存<檔案內容>的元件。

### <div id="file_thumbnail">縮圖 <path>(視窗內容\檔案總管\型態\檔案)</path></div>
是否要縮圖。
* 若要縮圖時，原圖比設定`解析度`小時 : 不縮圖。
* 若要縮圖時，原圖比設定`解析度`大時 : 待測，會依比例縮放嗎?

### <div id="file_thumbnail_resolution">解析度 <path>(視窗內容\檔案總管\型態\檔案\縮圖)</path></div>
縮圖至哪個尺寸。
* 1600*1200
* 1024*768
* 800*600
* 640*480
* 320*240

### <div id="color">調色盤 <path>(視窗內容)</path></div>
開啟調色盤，選擇顏色。
* 開啟介面
    * STD表單 / RWD表單

        ![調色盤]

        * 選擇器出現的位置 : 元件<內容區>下方，靠左顯示。但若右邊沒位置顯示不下時，改靠右顯示。
        * 當螢幕大小改變時，調色盤會依照目前螢幕大小跟著改變位置以及長寬，避免顯示不全(#7980)。
        * 若<輸入值>有值時，開啟選擇器後，會停在該顏色上。
        * 回傳值 : 6 碼  ex.紅色=FF0000。

* 挑選後，按下[Choose]會立即帶回。
